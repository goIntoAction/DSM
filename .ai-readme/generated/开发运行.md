## 签名配置

### 本地开发签名

调试版本使用自动调试签名，发布版本需要配置：

创建 `~/.gradle/gradle.properties` 或在命令行指定：

```properties
RELEASE_STORE_FILE=/path/to/keystore.jks
RELEASE_STORE_PASSWORD=your_store_password
RELEASE_KEY_PASSWORD=your_key_password
RELEASE_KEY_ALIAS=your_key_alias
```

### CI/CD 签名

GitHub Actions 使用环境变量配置签名：

```yaml
- name: Setup signing
  env:
    KEYSTORE_BASE64: ${{ secrets.KEYSTORE_BASE64 }}
  run: |
    echo $KEYSTORE_BASE64 | base64 -d > keystore.jks
    mkdir -p ~/.gradle
    cat > ~/.gradle/gradle.properties << EOF
    RELEASE_STORE_FILE=../keystore.jks
    RELEASE_STORE_PASSWORD=${{ secrets.KEYSTORE_PASSWORD }}
    RELEASE_KEY_PASSWORD=${{ secrets.KEY_KEY_PASSWORD }}
    RELEASE_KEY_ALIAS=${{ secrets.KEY_ALIAS }}
    EOF
```

## 开发服务器

### 连接 DSM

应用需要连接到 Synology DSM 服务器：

**支持的协议**：
- HTTP（开发调试）
- HTTPS（生产环境）

**开发配置**：
```xml
<!-- AndroidManifest.xml -->
<application
    android:usesCleartextTraffic="true"
    android:networkSecurityConfig="@xml/network_security_config"
    ...>
```

**网络安全配置**允许明文 HTTP 用于本地开发：
```xml
<!-- res/xml/network_security_config.xml -->
<network-security-config>
    <base-config cleartextTrafficPermitted="true">
        <trust-anchors>
            <certificates src="system" />
            <certificates src="user" />
        </trust-anchors>
    </base-config>
</network-security-config>
```

## 常见问题

### Q: 如何添加新的 API 接口？

1. 在 `data/api` 中创建 Retrofit 接口
2. 在 `di/ApiModule.kt` 中提供实例
3. 在 `data/model` 中定义响应模型
4. 在 `data/repository` 中创建 Repository

### Q: 如何调试网络请求？

网络日志在 Debug 模式自动启用，可在 Logcat 查看：
- 过滤器：`DsmApiClient`
- 级别：Debug

### Q: 如何处理自签名证书？

在设置中关闭 SSL 验证：
```kotlin
// SettingsManager
val checkSsl: StateFlow<Boolean>
```

或在开发时使用 HTTP 连接。
