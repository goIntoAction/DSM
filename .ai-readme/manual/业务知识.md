## 业务规则

### 会话管理规则

1. **登录凭据存储**：
   - sid、cookie、synoToken 加密存储在 DataStore
   - 主机地址 (host) 明文存储用于重连

2. **会话有效期**：
   - 默认会话有效期由 DSM 服务器配置决定
   - 检测到会话过期自动尝试重登录

3. **生物识别认证**（可选）：
   - 支持指纹/面部识别启动认证
   - 认证成功后进入登录页（非直接进入应用）

### 文件操作规则

1. **路径规范**：
   - 所有文件路径使用绝对路径
   - 路径必须以 `/` 开头
   - 特殊字符需要 URL 编码

2. **权限检查**：
   - 所有文件操作前检查用户权限
   - 权限不足时返回错误码 1006

3. **文件下载**：
   - 使用前台服务保证后台下载
   - 下载任务状态持久化到 Room 数据库

### 错误处理规则

1. **会话过期处理**：
   - 错误码 106/107/109/409 触发会话过期
   - 自动跳转到登录页

2. **网络重试**：
   - 网络错误最多重试 3 次
   - 使用指数退避策略（1s, 2s, 3s）

3. **文件传输**：
   - 上传/下载使用独立 OkHttpClient（300s 超时）
   - 不支持断点续传（依赖 DSM API 能力）

## DSM API 版本兼容

### API 版本差异

| API | DSM 6 | DSM 7 |
|-----|-------|-------|
| 照片 API | SYNO.Photo.* | SYNO.Foto.* |
| 认证 API | version 6 | version 7 |

**处理逻辑**：
```kotlin
val apiName = if (dsmVersion >= 7) "SYNO.Foto.Thumbnail" else "SYNO.Photo.Thumbnail"
```

### 版本检测

登录成功后自动检测 DSM 版本：
```kotlin
DsmApiHelper.updateDsmVersion(version)
```

## 安全考虑

### 1. 会话安全

- 使用 HttpOnly Cookie 存储 sid
- SynoToken 防止 CSRF 攻击
- 支持 HTTPS 加密通信

### 2. 本地存储

- 敏感信息 (sid, password) 加密存储
- 使用 Android Keystore 保护加密密钥

### 3. 网络安全

- 支持 SSL/TLS 加密
- 可配置跳过证书验证（用于自签名证书）
- 启用明文 HTTP（用于本地开发）
